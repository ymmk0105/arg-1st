# SPEC.md

## 概要
ブラウザのみで遊べる短編ARG。
GitHub Pagesで公開する静的Webサイトとして実装する。

---

## 技術要件

- HTML / CSS / JavaScript のみ
- サーバーサイド処理なし
- npm / ビルドツールなし
- GitHub Pagesで動作
- 相対パス使用
- スマホ対応
- localStorage 使用可
- 外部API禁止

---

## ページ構成

- index.html（トップ）
- intro.html（導入）
- case-file.html（ログ調査）
- archive.html（アーカイブ）
- hidden-log.html（隠しページ）
- ending.html（エンディング）

---

## 機能要件

### 1. ページ遷移
- リンクまたはJSで遷移
- 相対パスで実装

### 2. パスコード入力
- input + ボタン
- 正誤判定
- 不正解時のメッセージ表示

### 3. ヒント表示
- ボタン押下で表示
- 段階的ヒント（任意）

### 4. localStorage
- 進行状態を保存（任意）
- ページ解放フラグなど

### 5. 隠しリンク
- CSSまたは透明リンクで実装

---

## UI仕様

- フォントサイズ：最低16px以上
- 行間：読みやすさ重視
- 色：白ベース + 灰 + アクセント赤
- スマホでも崩れないレスポンシブ

---

## デザイン方針

- 古い記録サイト風
- シンプル
- テキスト主体
- 軽量

---

## UX設計

- 初見でも進める
- 詰まらない
- ヒントあり
- 一本道で迷子にならない

---

## 禁止事項

- 実在人物・企業使用
- 外部SNS誘導
- 個人情報入力
- 総当たり前提の謎
- 無理ゲー難易度

---

## GitHub Pages要件

- index.html がルートにある
- 絶対パス禁止
- すべて静的ファイル
- README.md に公開手順記載

---

## READMEに書く内容

- 公開手順
- 更新方法
- ディレクトリ構成
- 注意点

---

## 完了条件

- 最初から最後までプレイ可能
- 謎がすべて成立
- リンク切れなし
- スマホで閲覧可能
- GitHub Pagesで動作
